## 三、分层架构 & 调用链路

```
┌─────────────────────────────────────────────────────┐
│  uas-api (REST / WebSocket 接入层)                    │
│  Controller → 接收请求，调用 Service，返回 ApiResult  │
├─────────────────────────────────────────────────────┤
│  uas-core (核心业务层)                                │
│  Service (接口) → ServiceImpl (实现)                  │
│       ↓                                              │
│  Mapper (MyBatis-Plus BaseMapper) → Entity → DB      │
│       ↓                                              │
│  Redis (缓存 / 实时状态)                              │
│       ↓                                              │
│  Spring Event (内部事件总线)                          │
├─────────────────────────────────────────────────────┤
│  uas-common (公共模块)                                │
│  枚举、DTO、事件基类、ApiResult、工具类               │
├─────────────────────────────────────────────────────┤
│  uas-device-gateway (设备网关)                        │
│  MQTT → Parser → Redis Stream → Platform 消费        │
└─────────────────────────────────────────────────────┘
```

### 调用链路示例：探测遥测上报

```
MQTT 消息 → MqttMessageHandler → MqttPayloadParser
    → RedisStreamPublisher.publish(DetectTelemetryEvent)
    → PlatformStreamConsumer.consume()
    → DroneService.handleTelemetry()
        ├─ Redis 缓存活跃目标
        ├─ 闯入评估 (ZoneIntrusionEvaluator)
        ├─ 发布 DroneUpdatedEvent
        └─ 无人值守编排 (UnattendedOrchestrator)
            ├─ 自动告警 (AlertService)
            └─ 自动反制 (CounterService)
```

## 四、代码规范 & 模式

### 4.1 Controller 模式

```java
@RestController
@RequestMapping("/api/v1/{resource}s")
@RequiredArgsConstructor          // ← Lombok 构造注入
public class XxxController {

    private final XxxService xxxService;  // ← 注入 Service 接口

    @GetMapping                     // 列表
    public ApiResult<PageResult<Xxx>> list() { ... }

    @GetMapping("/{id}")           // 详情
    public ApiResult<Xxx> detail(@PathVariable String id) { ... }

    @PostMapping                    // 新增
    public ApiResult<Xxx> create(@RequestBody Xxx entity) { ... }

    @PutMapping("/{id}")           // 更新
    public ApiResult<Xxx> update(@PathVariable String id, @RequestBody Xxx entity) { ... }

    @DeleteMapping("/{id}")        // 删除
    public ApiResult<Void> delete(@PathVariable String id) { ... }
}
```

**关键约定：**
- URL 前缀统一: `/api/v1/`
- 返回类型统一: `ApiResult<T>` (code=200 成功)
- 分页用 `PageResult<T>`
- 用 `@RequiredArgsConstructor` + `private final` 注入依赖

### 4.2 Service 模式

```java
// 接口
public interface XxxService {
    List<Xxx> list();
    Xxx getById(String id);
    Xxx create(Xxx entity);
    Xxx update(Xxx entity);
    void delete(String id);
}

// 实现
@Slf4j
@Service
@RequiredArgsConstructor
public class XxxServiceImpl implements XxxService {

    private final XxxMapper xxxMapper;
    private final StringRedisTemplate redisTemplate;  // 按需

    @Override
    public List<Xxx> list() {
        return xxxMapper.selectList(null);
    }
}
```

**关键约定：**
- 接口 + 实现分离
- `@Service` + `@RequiredArgsConstructor`
- 日志用 `@Slf4j`
- 条件查询: `LambdaQueryWrapper<Xxx>` 做安全的条件拼接

### 4.3 Entity 模式

```java
@Data
@TableName("xxx_table")
public class Xxx {
    @TableId
    private String id;              // String 类型 ID

    // @TableId(type = IdType.AUTO) // 自增主键用这个
    // private Long id;

    private String name;
    private LocalDateTime createdAt;
}
```

### 4.4 Mapper 模式

```java
@Mapper
public interface XxxMapper extends BaseMapper<Xxx> {
    // 简单 CRUD 直接用 BaseMapper
}
```

### 4.5 统一响应

```java
ApiResult.ok(data);                    // { code: 200, msg: "success", data: ... }
ApiResult.fail(400, "参数错误");       // { code: 400, msg: "参数错误", data: null }
```

## 七、后端生成模式（新增功能的标准流程）

### 7.1 文件创建顺序

```
步骤 1: Entity     → uas-core/.../entity/Xxx.java
步骤 2: Mapper     → uas-core/.../mapper/XxxMapper.java
步骤 3: Service接口 → uas-core/.../service/XxxService.java
步骤 4: Service实现 → uas-core/.../service/impl/XxxServiceImpl.java
步骤 5: Controller → uas-api/.../controller/XxxController.java
```

### 7.2 处理模式速查表

| 场景 | 模式 |
|------|------|
| **数据库 CRUD** | Entity → Mapper(extends BaseMapper) → Service 接口 → ServiceImpl → Controller |
| **Redis 缓存** | `StringRedisTemplate` + `RedisKeys` 常量 + JSON 序列化 (`JsonUtils`) |
| **带条件查询** | `LambdaQueryWrapper<Entity>().eq(condition, Entity::getField, value)` |
| **事件发布** | 定义 Event 类 → `ApplicationEventPublisher.publishEvent()` → `@EventListener` 消费 |
| **Redis Stream 消息** | 定义 Event + StreamKeys → gateway 侧 `RedisStreamPublisher` → platform 侧 `PlatformStreamConsumer` |
| **WebSocket 推送** | `WebSocketHub.broadcast()` / `sendToUser()` |
| **无人值守自动化** | `UnattendedOrchestrator.process()` → 匹配规则 → 触发动作 |
| **分页返回** | `new PageResult<>(total, list)` 包装 |

### 7.3 生成顺序 Checklist

```
□ 1. 在 docs/database/schema.sql 中添加建表 DDL
□ 2. 在 uas-core/entity/ 创建实体类
□ 3. 在 uas-core/mapper/ 创建 Mapper 接口
□ 4. 在 uas-core/service/ 创建 Service 接口
□ 5. 在 uas-core/service/impl/ 创建 Service 实现
□ 6. 在 uas-api/controller/ 创建 Controller
□ 7. 如需事件: 在 uas-core/event/ 添加事件类
□ 8. 如需枚举: 在 uas-common/enums/ 添加枚举
□ 9. 如需新 Redis Key: 在 uas-common/constant/RedisKeys.java 添加
□ 10. 验证: 启动服务，SpringDoc 自动生成 API 文档
```
